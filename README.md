# Sugar Site

Бесплатный статичный сайт для отображения данных глюкозы с Lumiflex LinX.

## Что внутри

- `index.html` — страница
- `style.css` — стили (тёмная тема)
- `app.js` — загрузка CSV, метрики, графики (Chart.js)
- `data.csv` — данные (заглушка с тестовыми значениями)

## Метрики

- Среднее, SD, CV
- GMI (≈HbA1c) по формуле ADA
- TIR / TBR / TAR (3.0 / 3.9–10 / 13.9 ммоль/л)
- Графики: общий за период, профиль по часам (медиана + 25/75 перцентили)

## 1. Локальный запуск (проверить, что работает)

```
cd C:\Users\user\sugar-site
python -m http.server 8000
```
Открой http://localhost:8000 — должен отобразиться график на тестовых данных.

(Просто открыть `index.html` двойным кликом не сработает — `fetch("data.csv")` блокируется браузером по CORS для `file://`.)

## 2. Деплой на GitHub Pages (бесплатно)

1. Зарегистрируйся на github.com (если ещё нет).
2. Создай новый репозиторий: например `sugar`. Можно сделать его **приватным**, но тогда GitHub Pages работает только на платном тарифе. Для бесплатной публикации нужен **публичный** репо — тут и пригождается пароль (см. ниже).
3. Залей в репо все 4 файла из этой папки (`index.html`, `style.css`, `app.js`, `data.csv`).
4. В настройках репо: **Settings → Pages → Branch: main / root → Save**.
5. Через 1–2 минуты сайт будет доступен по адресу `https://<твой-логин>.github.io/sugar/`.

## 3. Пароль на сайт

Так как репо публичный, любой может зайти. Поставь пароль (хэш SHA-256, не банк, но отсечёт случайных).

1. Открой консоль браузера (F12 → Console) и выполни:
   ```js
   crypto.subtle.digest("SHA-256", new TextEncoder().encode("ТВОЙ_ПАРОЛЬ"))
     .then(b=>console.log([...new Uint8Array(b)].map(x=>x.toString(16).padStart(2,"0")).join("")))
   ```
2. Скопируй полученный хэш.
3. В `app.js` найди `PWD_SHA256: ""` → вставь хэш в кавычки.
4. Закоммить.

> ⚠️ Это **не настоящая** защита — кто умеет открывать DevTools, увидит `data.csv` напрямую. Если данные действительно чувствительны — нужен бэкенд. Но для отсечения случайных гостей хватит.

## 4. Парсер CSV под Lumiflex LinX

В `app.js` → `CONFIG.COL_TIME_CANDIDATES` и `COL_GLU_CANDIDATES` — список возможных названий колонок. Парсер выбирает первое совпадение.

После первого экспорта из приложения **открой свой `data.csv` в текстовом редакторе** и проверь шапку (первая строка). Если названий колонок нет в списке — добавь их в `app.js`.

Примеры реальных шапок CGM-приложений:
- `Time,Glucose (mmol/L)`
- `Datetime,Glucose Value`
- `Дата и время;Глюкоза`

Парсер понимает запятую, точку с запятой, табуляцию как разделитель.

## 5. Обновление данных

### Способ А — вручную (просто)

1. В Lumiflex LinX: экспорт CSV → отправь себе (AirDrop / почта / файлы).
2. Замени `data.csv` в репо (через web-интерфейс GitHub: Add file → Upload).
3. Сайт обновится в течение минуты.

### Способ Б — iOS Shortcut (1 касание)

Создай Shortcut «Update Sugar»:
1. **Получить файл из приложения** (Lumiflex LinX → CSV)
2. **Получить содержимое URL**: PUT `https://api.github.com/repos/<user>/<repo>/contents/data.csv`
   - Headers: `Authorization: Bearer <твой-PAT>`, `Accept: application/vnd.github+json`
   - Body (JSON): `{"message":"update","content":"<base64 файла>","sha":"<sha текущего файла>"}`
3. Запускай командой Siri.

Personal Access Token (PAT) с правом `repo` на свой репо: github.com/settings/tokens → Generate new (classic).

### Способ В — авто из Apple Health (если Lumiflex LinX в неё пишет)

Проверь в iPhone: **Health (Здоровье) → Браузер → Питание → Уровень глюкозы в крови**. Если там есть данные от Lumiflex — можно сделать Shortcut, который запускается по расписанию (через Personal Automation: каждый день в 23:00) и читает HealthKit → формирует CSV → пушит в GitHub. Это ноль касаний.

Если данных в HealthKit нет — этот путь закрыт.

## 6. Безопасность и приватность

- Данные глюкозы — медицинские, относятся к чувствительным.
- Публичный репо = публичные данные. Пароль на странице данные **не скрывает**.
- Если важна приватность: либо платный приватный GitHub Pages, либо хостинг с реальной авторизацией (Cloudflare Workers + KV, Vercel Functions). Я могу настроить.
